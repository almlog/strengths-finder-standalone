## 1. はじめに

### 1.1 本ツールについて

「メンバープロファイル分析」は、チームメンバーの強み（ストレングスファインダー）と性格（16Personalities/MBTI）を統合的に分析し、チームビルディングや人材配置を支援するWebアプリケーションです。

### 1.2 動作環境

| 項目 | 要件 |
|------|------|
| ブラウザ | Chrome, Edge, Firefox（最新版） |
| 画面サイズ | PC推奨（タブレット・スマホ対応） |
| インターネット | 初回アクセス時のみ必要 |

### 1.3 アクセス方法

以下のURLにアクセスしてください：

```
https://almlog.github.io/strengths-finder-standalone
```

## 2. アカウント登録・ログイン

### 2.1 新規登録

1. アクセスURLを開く
2. 「新規登録」ボタンをクリック
3. 会社メールアドレス（@altx.co.jp）を入力
4. パスワードを設定（8文字以上）
5. 「登録」ボタンをクリック
6. 登録完了後、自動的にログイン

**注意事項:**
- 会社ドメイン以外のメールアドレスでは登録できません
- パスワードは8文字以上で設定してください

### 2.2 ログイン

1. メールアドレスを入力
2. パスワードを入力
3. 「ログイン」ボタンをクリック

### 2.3 パスワードリセット

1. ログイン画面で「パスワードを忘れた方」をクリック
2. 登録済みメールアドレスを入力
3. 「リセットメールを送信」をクリック
4. メール内のリンクから新しいパスワードを設定

## 3. メンバー管理

### 3.1 メンバー一覧画面

ログイン後、最初に表示されるのがメンバー管理画面です。

**表示項目:**
- 社員番号
- 氏名
- 部署コード
- 役職
- Top5資質（アイコン表示）
- MBTIタイプ

**操作:**
- 行クリック: メンバー選択（分析用）
- 編集アイコン: メンバー情報編集
- 削除アイコン: メンバー削除

### 3.2 メンバー追加

1. 「メンバー追加」ボタンをクリック
2. 以下の情報を入力:

| 項目 | 必須 | 説明 |
|------|------|------|
| 社員番号 | ○ | 一意のID（例: 12345678） |
| 氏名 | ○ | フルネーム |
| 部署コード | ○ | 所属部署（例: 13D12345） |
| 役職 | - | 一般/リーダー/マネージャー/部長/カスタム |
| ステージ | - | S1/S2/S3/S4/CONTRACT/BP（マネージャーモード用） |

3. StrengthsFinder資質を入力（後述）
4. 16Personalitiesを入力（後述）
5. 「保存」ボタンをクリック

### 3.3 StrengthsFinder資質の入力

**Top5資質の登録手順:**

1. 「資質1（最も強い）」のドロップダウンを開く
2. 34資質から1つ選択
3. 同様に資質2〜5を順番に選択

**34資質一覧:**

| カテゴリ | 資質名 |
|---------|--------|
| 実行力 | 達成欲、アレンジ、信念、公平性、慎重さ、規律性、目標志向、責任感、回復志向 |
| 影響力 | 活発性、指令性、コミュニケーション、競争性、最上志向、自己確信、自我、社交性 |
| 人間関係構築力 | 適応性、運命思考、成長促進、共感性、調和性、包含、個別化、ポジティブ、親密性 |
| 戦略的思考力 | 分析思考、原点思考、未来志向、着想、収集心、内省、学習欲、戦略性 |

**ヒント:**
- StrengthsFinderは公式サイト（Gallup）で診断できます
- 診断結果のTop5をそのまま入力してください
- 資質の順番（1〜5）は重要です（分析に影響）

### 3.4 16Personalities（MBTI）の入力

**入力手順:**

1. 「16Personalities」セクションを開く
2. 診断結果のタイプを選択（例: INTJ）
3. アイデンティティを選択（A: 自己主張型 / T: 慎重型）

**16タイプ一覧:**

| グループ | タイプ | 名称 |

| 外交官 | INFP | 仲介者 |
| 外交官 | ENFJ | 主人公 |
| 外交官 | ENFP | 広報運動家 |
| 番人 | ISTJ | 管理者 |
| 番人 | ISFJ | 擁護者 |
| 番人 | ESTJ | 幹部 |
| 番人 | ESFJ | 領事官 |
| 探検家 | ISTP | 巨匠 |
| 探検家 | ISFP | 冒険家 |
| 探検家 | ESTP | 起業家 |
| 探検家 | ESFP | エンターテイナー |

**ヒント:**
- 16Personalitiesは https://www.16personalities.com/ja で無料診断できます
- MBTIは任意入力です（StrengthsFinderのみでも分析可能）
- MBTIを入力するとより詳細な分析が可能になります

### 3.5 カスタム役職の追加

標準役職（一般/リーダー/マネージャー/部長）以外の役職を追加できます。

1. メンバー編集画面で役職欄の「＋」ボタンをクリック
2. 役職名を入力（例: 「副事業部長」）
3. 色を選択（HEXコード）
4. アイコンを選択（crown/circle/star）
5. 「追加」をクリック

## 4. 個人分析

### 4.1 分析画面へのアクセス

1. メンバー一覧で分析したいメンバーをクリック
2. 「個人分析」タブを選択

### 4.2 プロファイルカード

選択したメンバーの統合分析結果が表示されます。

**表示項目:**

| 項目 | 説明 |
|------|------|
| 役割タイプ | 16パターンから自動推論（例: ビジョン構築者） |
| プロファイルサマリー | 4文構成の性格・強み説明 |

### 4.3 分析スコア

3つのスコアが表示されます（MBTI入力時）:

| スコア | 範囲 | 説明 |
|--------|------|------|
| 相性スコア | 0-100 | MBTI × Top5資質の適合度 |
| チーム適合度 | 0-100 | チームでの協働適性（Belbin理論） |
| リーダーシップ潜在力 | 0-100 | リーダーシップ資質（Bass理論） |

**スコアの見方:**

| 範囲 | 評価 |
|------|------|
| 85〜100 | 非常に高い |
| 70〜84 | 高い |
| 55〜69 | やや高い |
| 40〜54 | 標準 |
| 25〜39 | やや低い |
| 10〜24 | 低い |
| 0〜9 | 非常に低い |

**ツールチップ:**
各スコアにマウスを合わせると、計算の内訳と改善のヒントが表示されます。

### 4.4 資質レーダーチャート

34資質を4カテゴリに分類したレーダーチャートが表示されます。

- **実行力**: オレンジ系
- **影響力**: 赤系
- **人間関係構築力**: 青系
- **戦略的思考力**: 緑系

Top5資質は色付きで強調表示されます。

### 4.5 16Personalities詳細

MBTIタイプに基づく詳細情報が表示されます:

- タイプの特徴
- 強み・弱み
- 適した職種
- 相性の良いタイプ

### 4.6 SF-onlyモード

MBTIを入力していない場合、StrengthsFinderのみで分析されます:

- 34資質の特性に基づくプロファイル生成
- 7段階のスコア評価
- 20+種類の役割タイプ推論
- 「16Personalitiesを追加するとより詳細な分析が可能」のメッセージ表示

## 5. 部署分析

### 5.1 分析画面へのアクセス

1. 「部署分析」タブを選択
2. 分析したい部署をドロップダウンから選択

### 5.2 3カラムダッシュボード

部署分析は3つのパネルで構成されます:

#### 左パネル: 強みグループ分布（円グラフ）

4カテゴリの割合を視覚化:
- 実行力（%）
- 影響力（%）
- 人間関係構築力（%）
- 戦略的思考力（%）

#### 中央パネル: 頻出資質TOP10（棒グラフ）

部署内で最も多く保有される資質のランキング:
- 棒にマウスを合わせると保有者名が表示
- 保有率（%）も表示

#### 右パネル: チーム特性ナラティブ

自動生成されるチーム分析テキスト:

**タイトル例:**
- 「バランス型チーム」: 4カテゴリが均等
- 「実行力特化チーム」: 1カテゴリが40%以上
- 「実行力×戦略的思考力チーム」: 2カテゴリが各30%以上

**サマリー:**
TOP3資質とチーム特性に基づく文章

**頻出資質TOP5:**
保有率パーセンテージ付きリスト

**チームの可能性:**
3〜5項目の具体的な強み

## 7. 所有者分析

### 7.1 概要

特定の資質やMBTIタイプを持つメンバーを逆引き検索する機能です。

### 7.2 使い方

1. 「所有者分析」タブを選択
2. 検索条件を選択:
   - **資質で検索**: 34資質から1つ選択
   - **MBTIで検索**: 16タイプから1つ選択
3. 該当するメンバー一覧が表示

### 7.3 活用例

- 「戦略性」を持つメンバーを探したい
- 「ENTJ」タイプのメンバーを確認したい
- 特定資質の保有者数を把握したい

### 8.1 概要

来期の組織変更やプロジェクトチーム編成を事前にシミュレーションする機能です。

### 8.2 シミュレーション画面

1. 「シミュレーション」タブを選択
2. 画面構成:
   - **未割当プール**: まだグループに割り当てていないメンバー

### 8.3 グループの作成

1. 「グループ追加」ボタンをクリック
2. グループ名を入力（例: 「プロジェクトA」）

### 8.4 メンバーの配置

**ドラッグ&ドロップ操作:**
1. メンバーカードをドラッグ
2. 目的のグループにドロップ
3. リアルタイムで統計が更新

**メンバーカード表示:**
- 氏名
- Top5資質（アイコン）
- MBTIタイプ
- ステージ（マネージャーモード時）

### 8.5 グループ分析

各グループには以下の分析が表示されます:

| 項目 | 説明 |
|------|------|
| メンバー数 | グループ内の人数 |
| 相性スコア平均 | グループ内の平均相性 |
| チーム適合度平均 | グループの協働適性 |

| 強み分布 | 4カテゴリの円グラフ |

**保存:**
1. 「シナリオ保存」をクリック
2. シナリオ名を入力
3. ブラウザに保存される

**エクスポート:**
1. 「エクスポート」をクリック
2. JSONファイルがダウンロード
3. 他のPCや共有に使用可能

**インポート:**
1. 「インポート」をクリック
2. JSONファイルを選択
3. シナリオが読み込まれる

### 8.7 本番反映

シミュレーション結果を実際のメンバーデータに反映する機能:

1. 「本番反映」ボタンをクリック
2. 確認ダイアログで変更内容をプレビュー
3. 「反映する」をクリック

**注意:** 本番反映は元に戻せません。事前にエクスポートでバックアップを取ってください。

## 9. 勤怠分析

### 9.1 概要

楽楽勤怠システムからエクスポートしたXLSXファイルを分析し、勤怠違反や36協定超過をチェックする機能です。

### 9.2 ファイルのアップロード

1. 「勤怠分析」タブを選択
2. 楽楽勤怠から「出勤簿_日別詳細」をXLSX形式でエクスポート
3. 必要に応じて「分析オプション」を設定（後述）
4. ファイルをドラッグ&ドロップ、またはクリックして選択
5. 自動的に分析が開始

**対応ファイル:**
- 形式: XLSX（Excel形式）
- ファイル名例: `出勤簿_日別詳細_YYYYMMDDHHMMSS.xlsx`
- サイズ上限: 10MB

### 9.2.1 分析オプション

ファイルアップロード前に設定できるオプションがあります。

| オプション | デフォルト | 説明 |
|------------|------------|------|
| 今日の日付を含める | OFF（除外） | 当日の勤怠データを分析対象に含めるか |

**「今日を含める」トグルについて:**

- **OFF（今日を除外して分析）**: 推奨設定
  - 当日は勤務中で退勤打刻前の場合が多いため、「勤怠未入力」として誤検出されるのを防止
  - 業務時間中に分析する場合はこちらを推奨

- **ON（今日を含めて分析）**:
  - 当日の勤怠も分析対象に含める
  - 退勤後や翌日に分析する場合に使用
  - 退勤前でも違反判定される場合あり

### 9.3 分析結果の見方

#### サマリータブ

全体統計が表示されます:

| 項目 | 説明 |
|------|------|
| 対象従業員数 | 分析対象のユニーク従業員数 |
| 総出勤日数 | 全員の出勤日合計 |
| 総残業時間 | 36協定対象の残業時間合計 |
| 平均残業（月/人） | 従業員あたりの月平均 |
| 休日出勤日数 | 土日祝の出勤日数 |
| 違反件数 | 検出された違反の総数 |

**36協定アラート:**
残業時間が閾値を超えた従業員がリスト表示されます:

| レベル | 色 | 閾値 | 対応 |
|--------|-----|------|------|
| 注意 | 黄 | 35h超 | 上長報告 |
| 超過 | オレンジ | 45h超 | 36協定確認 |
| 警戒 | 赤 | 55h超 | 残業抑制指示 |
| 深刻 | 濃赤 | 65h超 | 残業禁止検討 |
| 重大 | 紫 | 70h超 | 親会社報告 |
| 危険 | 黒赤 | 80h超 | 医師面接指導 |
| 違法 | 黒 | 100h超 | 即時是正必須 |

**予兆アラート:**
月末までに45時間を超過する見込みの従業員が表示されます:
- 現在の残業時間
- 月末予測時間
- 超過見込み時間

#### 従業員別タブ

各従業員の詳細が表示されます:

| 項目 | 説明 |
|------|------|
| 社員番号 | 従業員ID |
| 氏名 | 従業員名（ホバーでTop5資質表示） |
| 部門 | 所属部門コード |
| 出勤日数 | 期間内の出勤日 |
| 残業時間 | 36協定対象時間（アラートアイコン付き） |
| 休日出勤 | 土日祝の出勤日数 |
| 遅刻/早退 | 遅刻・早退の回数 |
| 違反件数 | 検出された違反数 |

**StrengthsFinder連携:**
従業員名にマウスを合わせると、登録済みの場合はTop5資質がツールチップ表示されます。

#### 部門別タブ

部門ごとの集計:
- 従業員数
- 総残業時間
- 平均残業
- 違反件数

#### 違反一覧タブ

検出されたすべての違反が一覧表示されます:

| 違反タイプ | 説明 |
|-----------|------|
| 打刻漏れ | 出勤/退勤打刻がない |
| 休憩不足 | 法定休憩時間を満たしていない |
| 遅刻申請漏れ | 遅刻しているが申請がない |
| 早退申請漏れ | 早退しているが申請がない |
| 早出申請漏れ | 早出しているが申請がない |
| 時間有休打刻漏れ | 時間有休申請があるが打刻がない |
| 深夜休憩申請漏れ | 深夜勤務があるが休憩申請がない |
| 備考欄未記入 | 直行直帰等で備考が必要だが未入力 |
| 備考欄フォーマット警告 | 備考が5文字未満 |

### 9.4 8時出社カレンダー登録者への補正処理

**背景:**
- 従業員のデフォルト出社時間は **9:00** です
- しかし、楽楽勤怠のカレンダー設定で「8時〜」のスケジュールに登録されているメンバーが存在します
- システムはシート名から出社時間を特定するため、8時出社カレンダーのメンバーは9:00出社でも「1時間遅刻」と誤判定されてしまいます

**補正ロジック:**
以下の条件をすべて満たす場合、カレンダー設定の不整合による誤検出と判断し、遅刻違反としてカウントしません:
- シート名が8時出社カレンダー（「8時〜」等）である
- 「遅刻」が30分以下（実際は9:00〜9:30の出社）
- 実働時間が7時間30分以上
- 定時後に退勤している

**注意:** これは遅刻を許容するルールではなく、カレンダー登録の不整合を補正するための処理です。実際に遅刻している場合は別途対応が必要です。

### 9.5 CSV出力

分析結果をCSV形式でダウンロードできます:

1. 「CSV出力」ボタンをクリック
2. ファイルがダウンロード
3. Excelで開いて確認・加工可能

**出力形式:** UTF-8 BOM付き（Excel対応）

## 10. マネージャーモード

### 10.1 アクセス方法

URLに `?mode=manager` を追加:
```
https://almlog.github.io/strengths-finder-standalone?mode=manager
```

### 10.2 ステージマスタ設定

従業員のステージ（等級）ごとの原価設定を管理します。

1. 「設定」→「ステージマスタ」を開く
2. 各ステージの設定を入力:

| 項目 | 説明 | 例 |
|------|------|-----|
| 人件費合計 | 給与+交通費+残業見込み+賞与引当（円/月） | 450,000 |
| 経費率 | 部署経費+本社経費+営業口銭の平均（%） | 26 |
| 固定経費 | 契約社員・BP向けの固定費（円） | 50,000 |

**標準ステージ:**
- S1: 新人〜若手
- S2: 中堅
- S3: リーダー
- S4: マネージャー
- CONTRACT: 契約社員
- BP: ビジネスパートナー

### 10.3 単価情報管理

メンバーごとの売上単価を設定します。

1. メンバー編集画面を開く
2. 「単価情報」セクションで入力:

| 項目 | 説明 |
|------|------|
| 売上単価（月額） | 顧客への請求単価（円/月） |
| 契約単価（時給） | 契約社員・BPの時給（円/時） |

### 10.4 利益率計算ロジック

**正社員（S1-S4）:**
```
原価 = 人件費合計 + (人件費合計 × 経費率)
利益 = 売上単価 - 原価
利益率 = 利益 ÷ 売上単価 × 100
```

**契約社員:**
```
原価 = 契約単価 + 固定経費(50,000円) + (契約単価 × 26%)
```

**ビジネスパートナー:**
```
原価 = 契約単価 + 固定経費(40,000円) + (契約単価 × 9%)
```

### 10.5 利益率ダッシュボード

チーム全体の利益集計が表示されます:

| 項目 | 説明 |
|------|------|
| 総売上 | 全メンバーの月額売上合計 |
| 総原価 | 全メンバーの月額原価合計 |
| 総利益 | 売上 - 原価 |
| 利益率 | 総利益 ÷ 総売上 × 100 |
| 年間予測 | 月次 × 12ヶ月 |

**ステージ別内訳:**
S1 → S2 → S3 → S4 → CONTRACT → BP の順で表示

**部署フィルター:**
ドロップダウンで部署を選択すると、その部署のみの集計に切り替え

## 11. 設定・データ管理

### 11.1 テーマ切り替え

1. 画面右上のテーマアイコンをクリック
2. Light / Dark モードが切り替わる
3. 設定は自動保存される

### 11.2 データエクスポート

現在のデータをJSONファイルとして保存します。

1. 「設定」→「エクスポート」をクリック
2. JSONファイルがダウンロード
3. バックアップとして保管

**出力内容:**
- カスタム役職
- メンバー情報（資質・MBTI含む）
- ステージマスタ（マネージャーモード時）
- 単価情報（マネージャーモード時）

### 11.3 データインポート

JSONファイルからデータを復元します。

1. 「設定」→「インポート」をクリック
2. JSONファイルを選択
3. 競合解決ダイアログが表示された場合、戦略を選択:

| 戦略 | 説明 |
|------|------|
| Replace（全置換） | 既存データを削除し、インポートデータで置き換え |
| Add（新規のみ追加） | 重複しないメンバーのみ追加 |
| Merge（マージ&更新） | 重複メンバーは更新、新規は追加 |

### 11.4 データの自動保存

すべてのデータはブラウザのLocalStorageに自動保存されます。

**注意事項:**
- ブラウザのキャッシュをクリアするとデータが消えます
- 定期的にエクスポートでバックアップを取ってください
- 別のブラウザやPCではデータが共有されません

## 12. トラブルシューティング

### 12.1 ログインできない

| 症状 | 対処法 |
|------|--------|
| パスワードが間違っている | パスワードリセットを実行 |
| メールアドレスが登録されていない | 正しいメールアドレスか確認 |
| ドメインエラー | 会社メールアドレス（@altx.co.jp）を使用 |

### 12.2 データが消えた

| 原因 | 対処法 |
|------|--------|
| ブラウザキャッシュをクリア | バックアップからインポート |
| 別のブラウザでアクセス | 元のブラウザでエクスポートしてインポート |
| シークレットモード | 通常モードで使用 |

### 12.3 画面が正しく表示されない

| 症状 | 対処法 |
|------|--------|
| レイアウトが崩れる | ブラウザをリロード（F5） |
| グラフが表示されない | ブラウザを最新版に更新 |
| ダークモードが効かない | テーマ切り替えボタンを再度クリック |

### 12.4 勤怠分析でエラーが出る

| 症状 | 対処法 |
|------|--------|
| ファイルが読み込めない | XLSX形式か確認 |
| 分析結果が空 | 楽楽勤怠の正しいエクスポート形式か確認 |
| 従業員が少ない | シート構成を確認（プロジェクト別になっているか） |

### 12.5 その他の問題

解決しない場合は、以下の情報とともに開発者に連絡してください:

- 使用ブラウザとバージョン
- エラーメッセージ（スクリーンショット）
- 操作手順の詳細
