# 【WEBシステム掲載用】楽楽勤怠 データ仕様・運用総括マニュアル

本システムは「楽楽勤怠」から連携されたデータを基に稼働状況を分析します。正確な労務管理と給与計算のため、全ユーザーは以下の仕様と運用ルールを遵守してください。

---

## 1. データ連携と確定スケジュール
分析データの基準となる日時と締め処理のルールです。

*   **月次データの確定:**
    *   **締め切り:** 翌月第1営業日の **13:00**。
    *   **データ基準:** 上記日時のデータを基に確定計算が行われます。これ以降の修正は総務によるロック解除が必要となり、給与計算への影響が生じます。
*   **概算集計:**
    *   月末2営業日前に概算集計が実施されます。前日までの入力・申請を完了させてください。

## 2. 画面ステータスとアラート定義
システム上の表示色の意味と、解消必須のエラーについてです。

| 画面表示（背景色・印） | ステータス | 状態・対応 |
| :--- | :--- | :--- |
| **赤色** (背景) | **申請中** | 申請提出済み、決裁待ちの状態。 |
| **黒色** (承認印) | **承認済** | 最終承認者が承認完了した確定データ。 |
| **灰色** (背景) | **修正/取消中** | 承認済みデータに対し、修正または取消申請中の状態。 |
| **白色** (背景) | **システム登録** | 管理者が直接登録したデータ。 |
| **▲ 赤い三角** | **アラート** | **【重要】** 打刻漏れ等の不備あり。**この状態では月締め（出勤簿提出）ができません**。 |

---

## 3. 【重要】入力・申請の運用ルール（旧運用からの変更点）
システム仕様上、**「備考欄への入力」だけでは勤怠計算に反映されません**。以下の項目は必ず指定の処理を行ってください。

### A. 「申請書」提出が必須の項目（備考のみはNG）
| 項目 | 正しい処理 | システム上の挙動 |
| :--- | :--- | :--- |
| **深夜休憩** (22:00-05:00) | **「休憩時間修正申請」** (深夜休憩修正) | 自動計算されない深夜休憩時間を控除し、深夜割増を正しく計算します。 |
| **時差出勤** | **「時差出勤申請」** | 定時時刻を変更し、遅刻・早退判定を回避します。 |
| **携帯待機 / 宿直** | **「待機申請」「宿直申請」** | 手当計算の根拠データとなります。 |
| **看護・介護休暇** (時間単位) | **「時間子の看護休暇申請」**等＋打刻 | 年間の取得日数・時間数の管理を行います。 |

### B. 時間有休と中抜けの打刻ルール
時間単位年休を取得する際は、以下の**「申請＋打刻」のセット**が必須です。
*   **必須アクション:** 「時間有休申請」に加え、退社時に**「私用外出」**、戻り時に**「私用戻り」**の打刻を行う。
*   **理由:** 打刻がない場合、所定休憩時間（例:12:00-13:00）を正しく除外できず、有休残数が過剰に減算されたり、アラート対象となります。
*   **禁止事項:** 所定休憩時間を時間有休で埋めて、退社時間を繰り上げることはできません。

### C. 勤務形態別の残業計算トリガー
*   **客先常駐者:** 早出残業は、出勤簿の**「早出フラグ」に「1」**を入力することで計算対象になります。
*   **内勤者:** 「早出申請」および「残業申請」の提出・承認が必要です（フラグでは計算されません）。

## 4. 申請理由（備考）入力ガイドライン
承認者が即座に内容を把握でき、後からデータ分析を可能にするため、以下の**「【事由】＋【詳細】」**形式で統一してください。

**基本フォーマット：【事由】＋【詳細】**

| シチュエーション | 推奨記載例 | 注意事項 |
| :--- | :--- | :--- |
| **電車遅延** | `JR山手線遅延 20分` | 別途NI Collabo 360等で「遅延証明書」の提出が必要です。 |
| **直行・直帰** | `K社ビル（水道橋）面談のため` | 訪問先と業務目的を明記してください。 |
| **時間有休(中抜け)** | `11:00-14:00 私用（通院）` | 打刻漏れがないか確認してください。 |
| **打刻修正** | `入館証忘れのため` | 単なる「修正」ではなく理由を明記してください。 |
| **AltX残業** | `課会参加（録画確認対応）` | 具体的なタスク内容を記載してください。 |
| **慶弔休暇** | `弔事 祖母 12/8逝去 3日間` | 対象者、日程、日数の内訳を明記してください。 |
| **看護・介護休暇** | `長女看護 2日目` | 対象者と、何日目の取得かを明記してください。 |

## 5. データの修正フロー
承認状態によって修正手順が異なります。

1.  **承認前（申請中）:**
    *   申請一覧から対象を選び**「取り止め（取り下げ）」**を行い、正しい内容で再申請してください。
2.  **承認済み（確定後）:**
    *   **内容訂正:** 対象の申請を選び**「修正申請」**を行います。元のデータは「修正申請元」となり、新しい申請が承認されると置き換わります。
    *   **取り消し:** 申請自体を無効にする場合は**「取消申請」**を行います。
3.  **月締め後:**
    *   原則修正不可です。やむを得ない場合は総務への連絡とロック解除依頼が必要です。

---

## 6. 管理者・承認者向けチェックリスト
承認時は以下の観点で確認し、不備があれば「差戻し」を行ってください。

*   [ ] **申請と打刻の整合性:** 時間有休や中抜け申請があるのに、「私用外出/戻り」打刻が抜けていないか。
*   [ ] **法的基準の遵守:**
    *   労働時間が6時間超で45分、8時間超で60分の休憩が確保されているか。
    *   36協定の上限（月45時間・年360時間、特別条項適用時も月100時間未満）を超過していないか。
*   [ ] **健康管理:**
    *   月80時間を超える時間外労働は健康リスクが高まります。医師の面接指導などの措置を検討してください。
