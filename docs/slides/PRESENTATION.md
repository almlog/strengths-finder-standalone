<div class="columns">

### 人材配置の課題

### 労務管理の課題
- 勤怠違反の見落とし
- 36協定超過の把握が後手
- 月末にならないと状況がわからない

</div>

# 本ツールが解決すること

| 課題 | 解決策 |
|------|--------|
| 強みが見えない | **34資質の可視化**とスコアリング |
| チーム編成が感覚的 | **データに基づく相性分析**と適合度評価 |
| 組織変更の効果が不明 | **シミュレーション機能**で事前検証 |
| 勤怠違反の見落とし | **自動違反検出**（9種類） |
| 36協定超過の把握遅れ | **予兆アラート**で早期警告 |

# 部署分析機能

部署単位の強み傾向を3つのビューで可視化

| ビュー | 内容 |
|--------|------|
| **円グラフ** | 4カテゴリの分布（実行力・影響力・人間関係・戦略的思考） |
| **棒グラフ** | 頻出資質TOP10ランキング |
| **ナラティブ** | チーム特性の自動生成テキスト |

### チームタイプ自動判定
- **バランス型**: 4カテゴリが均等
- **特化型**: 1カテゴリが40%以上
- **複合型**: 2カテゴリが各30%以上

# チームシミュレーション機能

組織変更やプロジェクトチーム編成を**事前検証**

### 操作方法
1. グループを作成（最大10個）
2. メンバーをドラッグ&ドロップで配置
3. リアルタイムで分析結果を確認

### 確認できる指標
- グループごとの強み分布
- 相性スコア・チーム適合度の平均
- チーム特性ナラティブ

# 予兆アラート機能

**月末を待たずに**残業超過を予測して早期警告

### 仕組み
```
現在の残業ペース × 残り営業日 = 月末予測残業時間
```

### 表示内容
- 現在の残業時間
- 月末予測時間
- 超過見込み時間

<div class="warning">

**例**: 1月15日時点で25時間 → 月末予測52時間 → **45h超過見込み**

</div>

# StrengthsFinder 34資質（4カテゴリ）

| カテゴリ | 資質名 | 特徴 |
|---------|--------|------|
| **実行力** | 達成欲、責任感、規律性、目標志向 など | 物事を成し遂げる |

**合計34資質**のうち、診断でTop5を特定

| グループ | タイプ | 名称 | 特徴 |
|---------|--------|------|------|
| **分析家** | INTJ, INTP, ENTJ, ENTP | 建築家、論理学者、指揮官、討論者 | 戦略的思考 |

### アイデンティティ
- **A型（自己主張型）**: ストレス耐性高、自信がある
- **T型（慎重型）**: 完璧主義、自己改善志向

# 相性スコアの計算ロジック

MBTIタイプとTop5資質の整合性を評価

### 計算式
```
相性スコア = Σ(資質相性値 × 重み係数)
```

### 重み係数（Gallup研究に基づく）
| 順位 | 重み | 寄与率 |
|------|------|--------|
| TOP1 | 0.50 | 50% |
| TOP2 | 0.30 | 30% |
| TOP3 | 0.15 | 15% |
| TOP4-5 | 0.05 | 5% |

**ポイント**: TOP3で95%を占める設計

---

# チーム適合度の計算ロジック

**Belbin 9ロール理論**に基づくチーム協働適性評価

### 設計思想
> 内向型も外向型も等しく評価する

### 計算式
```
チーム適合度 = 基礎点(50) + Belbinロールスコア + 資質スコア
```

### Belbinロールの例
| ロール | 該当タイプ | チームへの貢献 |
|--------|-----------|--------------|
| プラント | INTP, INFP | 革新的アイデア創出 |
| コーディネーター | ENFJ, ENTJ | チーム目標の調整 |
| チームワーカー | ISFJ, ESFJ | チーム調和、対人調整 |

---

# リーダーシップ潜在力の計算ロジック

**Bass変革型リーダーシップ理論**に基づく評価

### 変革型リーダーシップの4要素
| 要素 | 説明 | 関連資質 |
|------|------|---------|
| 理想化影響 | ビジョンを示す | 最上志向、未来志向 |
| 鼓舞的動機付け | 意味を伝える | コミュニケーション、ポジティブ |
| 知的刺激 | 考えを促す | 着想、戦略性 |
| 個別配慮 | 個人を見る | 成長促進、個別化 |

### MBTIタイプ別スコア
NTJ/NFJ系（ENTJ, INTJ, ENFJ, INFJ）が高スコア

---

# データ保護の設計思想

<div class="highlight">

**基本方針**: 個人情報は外部サーバーに送信しない

</div>

### アーキテクチャ
```
┌─────────────────────────────────────┐
│          ユーザーのブラウザ           │
│  ┌─────────────┐  ┌──────────────┐  │
│  │ アプリケーション │  │ LocalStorage │  │
│  │   (React)    │←→│  (データ保存)  │  │
│  └─────────────┘  └──────────────┘  │
└─────────────────────────────────────┘
         ↓ 認証のみ
┌─────────────────────────────────────┐
│    Firebase Authentication          │
│    (認証情報のみ、メンバーデータなし)    │
└─────────────────────────────────────┘
```

---

# セキュリティ対策

| 対策 | 内容 |
|------|------|
| **ドメイン制限** | @altx.co.jp のメールアドレスのみ登録可能 |
| **パスワード要件** | 8文字以上 |
| **HTTPS通信** | GitHub Pages による暗号化通信 |
| **クライアントサイド処理** | データはブラウザ外に出ない |

### 勤怠データの取り扱い
- XLSXファイルはブラウザ内で処理
- **サーバーには送信されない**
- 画面を閉じるとメモリから消去

# 分析機能の使い方

1. メンバー一覧でメンバーをクリック
2. 「個人分析」タブでプロファイルを確認

### 部署分析
1. 「部署分析」タブを選択
2. ドロップダウンで部署を選択

### チームシミュレーション
1. 「シミュレーション」タブを選択
2. グループを作成
3. メンバーをドラッグ&ドロップ

# 勤怠分析の使い方

### Step 1: データ準備
楽楽勤怠から「出勤簿_日別詳細」をXLSX形式でエクスポート

### Step 2: アップロード
「勤怠分析」タブでファイルをドラッグ&ドロップ

### Step 3: 結果確認
| タブ | 内容 |
|------|------|
| サマリー | 全体統計、36協定アラート |
| 従業員別 | 個人ごとの詳細 |
| 部門別 | 部門ごとの集計 |
| 違反一覧 | 検出された違反リスト |

# FAQ

### Q1: StrengthsFinderの診断結果がないと使えませんか？
**A**: Top5資質は必須入力ですが、MBTIは任意です。資質のみでも基本的な分析が可能です（SF-onlyモード）。

### Q2: 他のPCやブラウザからデータを見られますか？
**A**: データはブラウザのLocalStorageに保存されるため、エクスポート/インポートで移行が必要です。

### Q3: 勤怠データはサーバーに送信されますか？
**A**: いいえ。XLSXファイルはブラウザ内で処理され、外部には一切送信されません。

### Q4: マネージャーモードは誰でも使えますか？
**A**: URLパラメータを知っていれば誰でもアクセス可能です。機密情報の取り扱いにご注意ください。
